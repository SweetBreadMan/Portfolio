                  퍼블리싱의 결과물은 기획서, 디자인을 담아 개발자에게, 나아가서 웹 사이트의 이용자에게 전하는 소통의 창구라고 생각합니다. 단순히 눈에 보이는 것만을 중요시 하지 않고 이용자가 쉽게 접근하고 원하는 정보를 빠르게 찾을 수 있도록 고민하며, 이용자의 경험을 최우선으로 생각하며 퍼블리싱을 해내가는 웹 퍼블리셔가 되겠습니다.

                  퍼블리싱 원칙과 기본기

                  폴더 구조와 경로를 체계적으로 정리하며, 파일명과 폴더명의 구분을 쉽게 하여 퍼블리싱을 효율적으로 수행합니다. 퍼블리싱을 하면서 수정, 보완이 원활하게 하기 위해서 적절한 주석을 달고 인덴트에 신경 써서 코드의 가독성을 높여 앞으로 협업하게 될 팀원들을 생각하며 세세하게 작업해 나가겠습니다.

                  체계적인 퍼블리싱

                  바로 퍼블리싱 작업에 들어가기 보다는 밴치마킹 사이트 물색, 와이어프레임, UI 디자인, 기능정의서, HTML 구조 설계를 먼저 진행해서 체계적인 퍼블리싱을 목표로 작업하고 있습니다. 이러한 단계를 거침으로써 프로젝트를 제작할 때 발생할 수 있는 불필요한 과정을 배제해서 효율적인 퍼블리싱 결과물을 만들도록 노력하고 있습니다.

                  하드코딩

                  기능성을 제외한 모든 작업은 하드코딩을 원칙으로 순수 CSS로 작업합니다. Position, Float를 사용한 기존방식부터 Flex와 Grid를 사용해 레이아웃 구조를 설계하고, 반복되는 코드는 최대한 압축, 요약해서 사용하며 불필요한 코드는 사용하지 않기 때문에 추후에 유지보수 및 수정이 필요할 때 손쉽게 작업 가능한 코드를 사용합니다.

                  항상 배우려는 자세

                  새로운 기술을 학습하는데에 두려움이 없고 기대감을 가지고 임합니다. 새로 알게 된 기술을 실제 프로젝트에 적용하고 의도한대로 작동할때 보람을 느끼며 작업을 합니다. 항상 배울 자세가 되어있으며 현재에 안주하지 않고 발전해나가는 웹 퍼블리셔가 되겠습니다.
